Addition & Subtraction

Addition combines quantities into a total. Subtraction finds the difference between them. Together they are the two most fundamental arithmetic operations — and the first ones computers were ever built to perform.

Addition

What is Addition?

Addition takes two (or more) numbers called addends and produces their sum. On the number line, adding n means moving n steps to the right.

a + b = sum
// example
3 + 5 = 8    12 + 7 = 19
Commutative

a + b = b + a  — order does not change the sum.

Associative

(a + b) + c = a + (b + c)  — grouping does not matter.

Identity

a + 0 = a  — adding zero leaves any number unchanged.

Closed in ℕ

The sum of two natural numbers is always a natural number.

In JavaScript

js
// basic addition
const a = 3;
const b = 5;
const total = a + b;   // 8

// sum of an array — loop accumulator
const numbers = [1, 2, 3, 4, 5];
let s = 0;
for (const n of numbers) s += n;   // 15

// commutativity check
console.assert(a + b === b + a);   // always true

// running total with for...of
let running = 0;
for (const n of numbers) running += n;   // 15
JS tip: for (const n of array) iterates every element. for (let i = 0; i < n; i++) iterates a range. Prefer for...of when you don't need the index — it's cleaner and works on any iterable.
Subtraction

What is Subtraction?

Subtraction finds the difference between a minuend and a subtrahend. On the number line it means moving left. Unlike addition, subtraction is not commutative — order matters.

a − b = difference
// example
9 − 4 = 5    20 − 13 = 7
Not commutative

a − b ≠ b − a (in general). Swapping gives the opposite sign.

Inverse of addition

a − b = a + (−b). Subtraction is just adding a negative.

Not closed in ℕ

5 − 8 = −3 which is not a natural number. We need integers ℤ.

Zero difference

a − a = 0 for any number a.

In JavaScript

js
// basic subtraction
const a = 9;
const b = 4;
const diff = a - b;   // 5

// not commutative
console.assert(a - b !== b - a);   // 5 !== -5

// subtraction as adding a negative
console.assert(a - b === a + (-b));   // true

// absolute difference — always non-negative
Math.abs(a - b);   // 5
Math.abs(b - a);   // 5  — same result

Algorithms

How Computers Add

At the hardware level, computers use binary addition — the same column-by-column carry algorithm you learned in school, but in base 2 instead of base 10. In JavaScript you can inspect bit representations with .toString(2).

Column addition (base 10)

js
// Add two non-negative integers digit by digit with carry.
function columnAdd(a, b) {
    const digits = [];
    let carry = 0;

    while (a > 0 || b > 0 || carry > 0) {
        const col = (a % 10) + (b % 10) + carry;
        digits.push(col % 10);
        carry = Math.floor(col / 10);
        a = Math.floor(a / 10);
        b = Math.floor(b / 10);
    }

    let result = 0, place = 1;
    for (const d of digits) { result += d * place; place *= 10; }
    return result;
}

// columnAdd(348, 275) === 623

Binary addition

js
const a = 5, b = 3;
(a).toString(2);         // "101"
(b).toString(2);         // "11"
(a + b).toString(2);     // "1000"  (= 8)

// two's complement: how CPUs handle subtraction
// a - b  ==  a + (~b + 1)  in fixed-width integers
function twosComplementSub(a, b, bits = 8) {
    const mask  = (1 << bits) - 1;
    const neg_b = ((~b) & mask) + 1;   // negate b
    return (a + neg_b) & mask;
}

// twosComplementSub(9, 4) === 5
Patterns in Code

Common Patterns

Accumulator pattern

js
// Sum 1 + 2 + 3 + … + n  (Gauss formula: n*(n+1)/2)
function sumToN(n) {
    return n * (n + 1) / 2;
}

// sumToN(100) === 5050

// same result with for loop
let s = 0;
for (let i = 1; i <= 100; i++) s += i;   // 5050

Change calculation

js
function makeChange(priceCents, paidCents) {
    const change = paidCents - priceCents;
    if (change < 0)
        throw new Error("Not enough money paid");
    return change;
}

// work in integer cents to avoid float rounding
// makeChange(350, 500) === 150  (= $1.50 change)
Floating point warning: 0.1 + 0.2 in JavaScript gives 0.30000000000000004 due to binary floating-point representation — the same as every other language using IEEE 754. For money, always work in integer cents or use a dedicated decimal library.

Safe money arithmetic (integer cents)

js
// floating-point imprecision
const x = 0.1 + 0.2;
// x !== 0.3  (binary float rounding)

// solution: store amounts as integer cents
const price  = 10;   // $0.10
const added  = 20;   // $0.20
const total  = price + added;   // 30 ✓  ($0.30)

// display with two decimal places
`$${(total / 100).toFixed(2)}`;   // "$0.30"
Summary

Key Takeaways

a + b = b + a

Addition is commutative. Subtraction is not.

a − b = a + (−b)

Subtraction is the inverse of addition.

for (const n of arr)

JavaScript's idiomatic loop for iterating arrays — use it for accumulators.

Use integer cents for money

Float arithmetic is imprecise — store money as integer cents.
