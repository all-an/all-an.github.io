Fractions & Decimals

A fraction represents a part of a whole. A decimal is another way to write the same idea — shifting between the two is a core skill that underpins percentages, ratios, probability, and most of the maths that follows.

Fractions

What is a Fraction?

A fraction a / b (b ≠ 0) has a numerator a on top and a denominator b on the bottom. It means: divide a into b equal parts and take a of them.

¾  →  numerator = 3,  denominator = 4
// value: 3 out of 4 equal parts → 0.75
Proper

|num| < den  e.g. 3/4

Improper

|num| ≥ den  e.g. 7/4

Mixed number

1¾ = 1 + 3/4 = 7/4

Equivalent

1/2 = 2/4 = 3/6 = …

Unit fraction

numerator = 1  e.g. 1/7

Simplest form

GCD(num, den) = 1

Greatest Common Divisor (GCD)

To reduce a fraction to its simplest form you need the Greatest Common Divisor — the largest integer that divides both numerator and denominator exactly, leaving no remainder. Dividing both by their GCD scales the fraction down uniformly, so its value stays identical but the numbers are as small as possible.

GCD(12, 8) = 4  →  12/8 ÷ 4/4 = 3/2
GCD(6, 4) = 2  →  6/4 ÷ 2/2 = 3/2
GCD(9, 3) = 3  →  9/3 ÷ 3/3 = 3/1 = 3
// a fraction is in simplest form when GCD(num, den) = 1
Why does dividing by GCD work?
Any common divisor d means num = d·p and den = d·q for some integers p and q. Cancelling d gives p/q, which has the same value. The greatest common divisor cancels as much as possible in one step, leaving a numerator and denominator that share no further factor — i.e. simplest form.

The Euclidean Algorithm

The fastest way to find GCD is the Euclidean algorithm: repeatedly replace the larger number with the remainder of dividing the two numbers, until the remainder is zero. The last non-zero value is the GCD.

GCD(48, 18)
48 = 2 × 18 + 12  →  GCD(18, 12)
18 = 1 × 12 + 6  →  GCD(12, 6)
12 = 2 × 6 + 0  →  remainder = 0, answer = 6
// so 48/18 simplifies to 8/3

Each step the numbers get strictly smaller, so the algorithm always terminates. It runs in O(log min(a, b)) steps — extremely fast even for large numbers.

In JavaScript

js
// Euclidean algorithm — swap (a, b) with (b, a % b) until b = 0
function gcd(a, b) {
    while (b) { [a, b] = [b, a % b]; }
    return a;  // last non-zero remainder
}

// trace: gcd(48, 18) → gcd(18, 12) → gcd(12, 6) → gcd(6, 0) → 6
console.log(gcd(48, 18));  // 6
console.log(gcd(12, 8));  // 4
console.log(gcd(7,  5));  // 1 — already coprime, no reduction possible

// simplify: divide both parts by their GCD (use Math.abs for negative numerators)
function simplify(num, den) {
    const g = gcd(Math.abs(num), Math.abs(den));
    return [num / g, den / g];
}

console.log(simplify(6, 4));   // [3, 2]  — GCD was 2
console.log(simplify(3, 9));   // [1, 3]  — GCD was 3
console.log(simplify(10, 4));  // [5, 2]  — GCD was 2
console.log(simplify(7, 5));   // [7, 5]  — already simplest, GCD = 1

Operations

Fraction Arithmetic

Addition and subtraction need a common denominator. Multiplication and division work directly on numerators and denominators.

a/b + c/d = (a·d + c·b) / (b·d) then simplify
a/b × c/d = (a·c) / (b·d) then simplify
a/b ÷ c/d = (a·d) / (b·c) then simplify
// example: 1/2 + 1/3 = 3/6 + 2/6 = 5/6
js
// add fractions → simplify result
function addFractions(n1, d1, n2, d2) {
    return simplify(n1 * d2 + n2 * d1, d1 * d2);
}
console.log(addFractions(1, 2, 1, 3));  // [5, 6]

// multiply fractions
function mulFractions(n1, d1, n2, d2) {
    return simplify(n1 * n2, d1 * d2);
}
console.log(mulFractions(2, 3, 3, 4));  // [1, 2]

// divide fractions → multiply by reciprocal
function divFractions(n1, d1, n2, d2) {
    return simplify(n1 * d2, d1 * n2);
}
console.log(divFractions(3, 4, 3, 8));  // [2, 1]  →  2
Decimals

What are Decimals?

A decimal is a base-10 notation for a number that is not a whole integer. Each digit after the decimal point represents a power of 1/10.

3.14 = 3 + 1/10 + 4/100
0.75 = 7/10 + 5/100 = 75/100 = 3/4
// terminating decimal ↔ fraction with denominator 2ⁿ × 5ⁿ
Terminating

0.25, 0.5, 0.125 — finite digits

Repeating

0.333…, 0.142857142857… — pattern repeats

Irrational

π, √2 — infinite, non-repeating

In JavaScript

js
// rounding to n decimal places
function roundTo(n, places) {
    const factor = 10 ** places;
    return Math.round(n * factor) / factor;
}
console.log(roundTo(3.14159, 2));  // 3.14
console.log(roundTo(1 / 3, 4));   // 0.3333

// floating-point caution
console.log(0.1 + 0.2);           // 0.30000000000000004 !
console.log(roundTo(0.1 + 0.2, 2)); // 0.3  ✓
Conversions

Fractions ↔ Decimals ↔ Percentages

These three forms represent the same value. Being fluent in converting between them is essential for mental arithmetic and for working with data.

Fraction → Decimal: divide num ÷ den
Decimal → Fraction: use place value, then simplify
Fraction → Percent: (num / den) × 100
// 3/4 = 0.75 = 75%
js
// fraction → decimal
console.log(3 / 4);    // 0.75

// fraction → percent
function toPercent(num, den) {
    return (num / den) * 100;
}
console.log(toPercent(3, 4));   // 75
console.log(toPercent(1, 3));   // 33.333…

// decimal → fraction (terminating)
// 0.75 → 75 / 100 → simplify
function decimalToFraction(d, precision = 1000000) {
    const num = Math.round(d * precision);
    return simplify(num, precision);
}
console.log(decimalToFraction(0.75));  // [3, 4]
console.log(decimalToFraction(0.1));   // [1, 10]

Summary

Key Takeaways

Simplify with GCD

Divide both num and den by GCD(num, den).

Common denominator

Required for addition/subtraction. Use LCM for efficiency.

Multiply/divide directly

No common denominator needed. Divide: multiply by the reciprocal.

Fraction → %

(num / den) × 100. Percent is just per-hundred.

Float caution in JS

0.1 + 0.2 ≠ 0.3. Round after arithmetic when precision matters.

Repeating decimals

1/3 = 0.333… — not exactly representable in binary floating point.
