Immutability / const

Immutability means a value cannot be reassigned or mutated after it is created, which makes program state easier to reason about.
Languages expose this with bindings like const, final, or val, though some lock the reference while others also protect the object itself.
The result is safer code with fewer accidental side effects, especially in concurrent and functional styles.

// Java
final int maxRetries = 3;      // cannot be reassigned
final String name = "Allan";   // reference stays fixed
var message = name + " ok";    // derived value from immutable data
