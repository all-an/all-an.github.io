Type casting / coercion

Casting changes how a value is treated by the type system, while coercion is an automatic conversion the language performs for you.
Some conversions are safe and widening, while others are narrowing and may lose information or fail at runtime.
In Java, numeric coercion can happen implicitly, but explicit reference or narrowing conversions use cast syntax like (int) or (String).

// Java
double price = 9.99;
int rounded = (int) price;   // explicit narrowing cast

int count = 5;
double total = count;        // implicit widening coercion

System.out.println(rounded); // 9
System.out.println(total);   // 5.0
