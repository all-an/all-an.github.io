EC2

Elastic Compute Cloud
Compute

What it is

Amazon EC2 gives you resizable virtual machines — called instances — that you rent by the second. It is the original AWS compute service and the foundation many others are built on. You pick how much CPU, memory, storage, and networking you want, launch in minutes, and turn it off when you are done.

You choose four things when you launch: an AMI (the disk image / operating system), an instance type (the hardware size, e.g. t3.micro), a key pair (for SSH login), and a security group (a virtual firewall). Everything else has sensible defaults.

Core concepts to know

  • AMI — Amazon Machine Image: the template (OS + pre-installed software) an instance boots from.
  • Instance type — the size and family: t (burstable), m (general), c (compute), r (memory), etc.
  • Key pair — a public/private key used to SSH in. AWS keeps the public half; you download the private .pem once.
  • Security group — stateful firewall rules controlling inbound/outbound traffic (e.g. allow port 22 from your IP).
  • EBS volume — the persistent disk attached to the instance; survives stop/start, deleted on terminate unless told otherwise.
  • Elastic IP — a static public IPv4 address you can keep across restarts.

Useful AWS CLI commands

Copy, then swap the placeholder IDs (ami-…, i-…, sg-…) for your own.

List your instances (id, state, type, public IP) as a clean table
aws ec2 describe-instances \
  --query "Reservations[].Instances[].{ID:InstanceId,State:State.Name,Type:InstanceType,IP:PublicIpAddress}" \
  --output table
Launch a single t3.micro instance
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t3.micro \
  --key-name my-key \
  --security-group-ids sg-0123456789abcdef0 \
  --count 1
Find the latest Amazon Linux 2023 AMI id for your region
aws ssm get-parameters \
  --names /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64 \
  --query "Parameters[0].Value" --output text
Stop, start, and terminate an instance
aws ec2 stop-instances      --instance-ids i-0123456789abcdef0
aws ec2 start-instances     --instance-ids i-0123456789abcdef0
aws ec2 terminate-instances --instance-ids i-0123456789abcdef0
Open SSH (port 22) to your current IP only
aws ec2 authorize-security-group-ingress \
  --group-id sg-0123456789abcdef0 \
  --protocol tcp --port 22 --cidr $(curl -s https://checkip.amazonaws.com)/32
SSH into the instance (uses your downloaded key)
ssh -i my-key.pem ec2-user@<public-ip>

Console: where to click

In the AWS Management Console, the buttons and menu items to follow:

Search bar→ EC2→ Instances→ Launch instances — create a new VM
EC2→ Instances→ (select instance)→ Connect→ EC2 Instance Connect — browser SSH, no key file needed
(select instance)→ Instance state→ Stop / Start / Terminate
EC2→ Network & Security→ Security Groups→ Edit inbound rules — open/close ports
EC2→ Network & Security→ Key Pairs→ Create key pair

Hints & tips

⚡ Stop ≠ Terminate. Stop shuts the VM down but keeps the EBS disk (and your data) — you still pay a little for storage. Terminate deletes the instance and, by default, its root disk. There is no undo.
⚡ Check the Region first. Instances are per-Region. If your list looks empty, look at the Region menu in the top-right corner of the console — you are probably in the wrong one.
⚡ Prefer EC2 Instance Connect or SSM Session Manager over opening port 22 to the world. They give you a shell without a public SSH port — far safer. Never set an inbound rule to 0.0.0.0/0 on port 22.
⚡ Free tier: t2.micro / t3.micro get 750 hours/month for the first 12 months. Pick those while learning to avoid surprise bills.
⚡ Watch for the stopped-but-billing trap: a stopped instance is free for compute, but its attached EBS volume and any Elastic IP not associated with a running instance still cost money. Release unused Elastic IPs.
⚡ Tag everything. Add a Name tag at launch so the console list is readable, and use tags for cost tracking later.
