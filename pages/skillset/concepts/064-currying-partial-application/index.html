Currying / partial application

Currying turns a function of several arguments into a chain of one-argument functions: instead of f(a, b) you call f(a)(b), and f(a) on its own is already a perfectly good function.
Partial application is the payoff: fix some arguments now and leave the rest for later, producing a specialized function you can name, pass around, and reuse.
Scala makes this first-class — a function can be declared with several parameter lists, and any ordinary multi-argument function can be curried on demand with .curried. Here add(a)(b) is curried so that add(10) becomes a reusable "add ten".

// Scala
// A curried function is written as several parameter lists. Supplying only the
// first list returns a function that is still waiting for the rest.
def add(a: Int)(b: Int): Int = a + b

// Partial application: fix the first argument and get back a new function.
// `add(10)(_)` leaves the second parameter open, yielding an Int => Int.
val add10: Int => Int = add(10)(_)

println(add10(5))   // 15
println(add10(1))   // 11
println(add(3)(4))  // 7  — or supply both lists at once

// Any uncurried function can be curried on demand with `.curried`.
val multiply: (Int, Int) => Int = (x, y) => x * y
val triple: Int => Int = multiply.curried(3)
println(triple(5))  // 15
