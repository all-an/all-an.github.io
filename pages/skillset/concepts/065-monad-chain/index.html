Monad / chain

A monad is a type that wraps a value and knows how to chain steps together with flatMap (also called bind or >>=). Each step produces a new wrapped value, and the monad decides what threading them together means.
Java ships several monads in its standard library — Optional, Stream, and CompletableFuture all chain with flatMap. This uses Optional: the moment a step returns empty (a failure), the rest of the chain is skipped and empty becomes the result — no manual if (failed) return at every step.
Each flatMap binds the unwrapped value of one step into the next; an empty anywhere short-circuits the remaining steps. That threading rule is the monad doing the work.

// Java
// Optional is a monad: it has flatMap (a.k.a. bind / >>=) and map, so an
// empty short-circuits the rest of the chain.
static Optional<Integer> parse(String text) {
  try {
    return Optional.of(Integer.parseInt(text));
  } catch (NumberFormatException e) {
    return Optional.empty();
  }
}

static Optional<Integer> divide(int a, int b) {
  return b == 0 ? Optional.empty() : Optional.of(a / b);
}

// Nested flatMap: bind each step to the next. An empty anywhere stops the chain.
static Optional<Integer> chained(String a, String b) {
  return parse(a).flatMap(x -> parse(b).flatMap(y -> divide(x, y)));
}

System.out.println(chained("84", "2")); // Optional[42]
System.out.println(chained("84", "0")); // Optional.empty
