Fiber / green thread

A fiber or green thread is a lightweight unit of execution that is cheaper to create than a traditional operating-system thread.
In modern Java, Project Loom provides this idea as virtual threads, so code can keep the direct thread-per-task style without paying the old platform-thread cost for every task.
The main benefit is scale and simpler blocking code, not a different shared-memory model by itself.

// Java
// Start two lightweight tasks; each gets its own virtual thread.
Thread first = Thread.startVirtualThread(() -> {
  System.out.println("first task");
});

Thread second = Thread.startVirtualThread(() -> {
  System.out.println("second task");
});

// The main thread joins both so execution does not continue too early.
first.join();
second.join();
System.out.println("all done");
