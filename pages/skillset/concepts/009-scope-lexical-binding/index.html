Scope / lexical binding

Scope decides where a name can be seen, and lexical binding means that visibility is determined by where code is written, not by who calls it at runtime.
Inner blocks and functions can read names from outer scopes, which is what makes closures work.
JavaScript uses lexical scope, so a function keeps access to the variables that were in scope when the function was defined.

// JavaScript
function makeCounter() {
  let count = 0;

  return function () {
    count += 1;
    return count;
  };
}

const counter = makeCounter();

counter(); // 1
counter(); // 2
