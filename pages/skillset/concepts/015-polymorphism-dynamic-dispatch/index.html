Polymorphism / dynamic dispatch

Polymorphism — Greek for "many forms" — is the OOP pillar that lets one call site talk to many concrete types through a shared parent type or interface.
In Java that pillar is implemented by dynamic dispatch: when you call a.speak() on a reference of static type Animal, the JVM looks up the actual subclass at runtime and invokes that subclass's override, not the compile-time parent's method.
Below, the same loop walks an Animal[] containing mixed subtypes, and each element answers in its own voice — the loop does not know or care which subclass it is holding.

// Java
class Animal {
  String speak() { return "..."; }
}

// Each subclass overrides speak() to return its own sound.
// @Override is documentation + a compile-time guard that the parent really has this method.
class Dog extends Animal {
  @Override String speak() { return "woof"; }
}
class Cat extends Animal {
  @Override String speak() { return "meow"; }
}
class Cow extends Animal {
  @Override String speak() { return "moo"; }
}

// The compile-time type of every element is Animal — the JVM has no idea at compile
// time which subclass will actually be in each slot.
Animal[] herd = { new Animal(), new Dog(), new Cat(), new Cow() };

// Yet calling a.speak() picks the correct override at RUNTIME for each object.
// This runtime selection is "dynamic dispatch" — the heart of OOP polymorphism.
for (Animal a : herd) {
  System.out.println(a.getClass().getSimpleName() + " says " + a.speak());
}
