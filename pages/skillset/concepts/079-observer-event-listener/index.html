Observer / event listener

The Observer pattern lets one object — the subject (or observable) — notify many others — the observers (or event listeners) — whenever its state changes, without knowing anything about them beyond a shared interface. Observers subscribe to the subject; when something happens, the subject pushes the new value to each one. This is a push model and it keeps the two sides loosely coupled: new listeners can be added without touching the subject.
It is one of the original Gang of Four patterns (1994) and underlies UI event listeners (Java Swing, the DOM), Spring's @EventHandler, C# event/delegate, Qt's signal/slot, and — taken to its asynchronous, stream-based extreme — the reactive Observable (RxJava/RxJS) and ZIO's ZStream. This is a runnable Maven project; see the README.md in this concept's folder in the cloned repository.

The observer is just an interface with a callback. Anything implementing it can listen:

// The observer / event listener — the subject knows listeners only through this.
interface TemperatureObserver {
  void onTemperatureChanged(double celsius);   // pushed the new value
}

The subject keeps a list of observers, exposes subscribe / unsubscribe, and broadcasts every change. It depends only on the interface, never on concrete listeners:

class WeatherStation {
  private final List<TemperatureObserver> observers = new ArrayList<>();
  private double celsius;

  void subscribe(TemperatureObserver o)   { observers.add(o); }
  void unsubscribe(TemperatureObserver o) { observers.remove(o); }

  // Update state and push it to everyone in one operation.
  void setTemperature(double celsius) {
    this.celsius = celsius;
    // Iterate a copy so an observer may unsubscribe from inside its callback.
    for (TemperatureObserver o : new ArrayList<>(observers)) {
      o.onTemperatureChanged(celsius);
    }
  }
}

Observers subscribe and react independently. Because the interface has a single method, a lambda works as a listener too — and unsubscribing stops the notifications:

WeatherStation station = new WeatherStation();
station.subscribe(new Display("Phone"));
station.subscribe(new Display("Window unit"));

// A lambda is a perfectly good observer.
station.subscribe(c -> { if (c > 30) System.out.println("Heat warning! " + c + "°C"); });

station.setTemperature(22.5);     // every subscriber reacts
station.unsubscribe(phone);       // phone opts out...
station.setTemperature(31.0);     // ...and is silent; the rest still react

The test injects a recording observer to prove the three guarantees — everyone is notified, the new value is delivered, and unsubscribing stops events:

@Test
void unsubscribedObserverStopsReceivingEvents() {
  WeatherStation station = new WeatherStation();
  RecordingObserver staying = new RecordingObserver();
  RecordingObserver leaving = new RecordingObserver();
  station.subscribe(staying);
  station.subscribe(leaving);

  station.setTemperature(10.0);          // both hear this
  station.unsubscribe(leaving);
  station.setTemperature(20.0);          // only `staying` hears this

  assertEquals(List.of(10.0, 20.0), staying.received);
  assertEquals(List.of(10.0), leaving.received);   // stopped after unsubscribe
}
