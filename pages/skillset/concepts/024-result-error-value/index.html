Result / error value

A result type represents either a successful value or an error value in the type system, instead of signaling failure with exceptions or nulls.
It forces the caller to handle both cases explicitly, which makes control flow around failure more visible.
Java has no standard built-in Result type, but TypeScript expresses it cleanly as a discriminated union — testing the tag narrows the type, so each branch can only touch its own field.

// TypeScript
// A Result is a discriminated union: a success carrying a value, OR a failure
// carrying an error. The literal `ok` field is the tag the compiler narrows on.
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };

// parseAge returns a value on success or an error message on failure — the
// failure is an ordinary return value, not a thrown exception or a null.
function parseAge(text: string): Result<number, string> {
  const n = Number(text);
  if (!Number.isInteger(n) || n < 0) return { ok: false, error: `invalid age: '${text}'` };
  return { ok: true, value: n };
}

const r = parseAge("42");

// Checking `.ok` narrows the union: `.value` is reachable only here, and
// `.error` only in the else branch — both cases must be handled.
if (r.ok) {
  console.log(`age is ${r.value}`);
} else {
  console.log(`error: ${r.error}`);
}
