Operator overloading

Operator overloading lets user-defined types give meaning to built-in operators like +, -, or ==.
Instead of writing a named method call, code can use familiar symbols while the language redirects the operation to custom logic.
It can make abstractions feel natural, but if used carelessly it also hides behavior behind syntax that looks simple.

// Scala
case class Vector(x: Int, y: Int) {
  // `+` is just a method name, so it can be defined like any other
  def +(other: Vector): Vector = Vector(x + other.x, y + other.y)
}

val a = Vector(2, 3)
val b = Vector(4, 1)
val sum = a + b   // Vector(6, 4)
