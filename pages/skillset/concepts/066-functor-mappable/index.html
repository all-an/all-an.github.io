Functor / mappable

A functor is any type that has map: it applies a plain function A -> B to the value(s) inside a container without unwrapping them, and keeps the container's shape. You hand map an ordinary function and it does the reaching-in for you.
Java ships several functors in its standard library — Optional and Stream both expose map — so the same function maps over either one. Over an Optional, map runs the function if a value is present and leaves empty untouched; over a Stream it runs the function on each element and returns a stream of the same length.
How does this differ from a monad? A functor's map takes a plain function (A -> B); a monad's flatMap takes a wrapping function (A -> F<B>). map is the simpler of the two, and every monad is also a functor.

// Java
// One plain function, A -> B. It knows nothing about Optional or Stream — that
// is the point: a functor lets you reuse plain functions over wrapped values.
Function<Integer, Integer> square = n -> n * n;

// Optional is a functor: map runs square inside the box if a value is present,
// and preserves "empty" untouched if it is not.
Optional<Integer> present = Optional.of(6).map(square);            // Optional[36]
Optional<Integer> absent  = Optional.<Integer>empty().map(square); // empty

// Stream is the same functor idea over many values: map applies square to each
// element and preserves the structure (a stream of the same length).
List<Integer> squared = Stream.of(1, 2, 3, 4)
    .map(square)
    .collect(Collectors.toList());                                // [1, 4, 9, 16]

// Functor composition law: mapping f then g equals mapping (g after f) once.
Optional<Integer> twoMaps = Optional.of(6).map(square).map(plusOne);   // Optional[37]
Optional<Integer> oneMap  = Optional.of(6).map(square.andThen(plusOne)); // Optional[37]
