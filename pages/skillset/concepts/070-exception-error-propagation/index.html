Exception / error propagation

A thrown exception propagates up the call stack: it unwinds each frame that does not handle it, until a matching catch clause is found. Java is the canonical home of this model — try/catch/finally with a typed exception hierarchy.
Here parsePositive (the deepest frame) throws, total (the middle frame) does not catch, so the exception passes straight through it to the try in main. The matching catch is chosen by type, most specific first: NumberFormatException is a subclass of IllegalArgumentException, so it must be listed before the broader type or Java rejects it as unreachable.
The finally block runs on every exit — after a normal return and after a caught exception — which is why it is the place for cleanup like closing files or releasing locks.

// Java
// The deepest frame. Throws two different exception types.
static int parsePositive(String text) {
  int n = Integer.parseInt(text);      // throws NumberFormatException on non-numbers
  if (n <= 0) {
    throw new IllegalArgumentException("not positive: " + n);
  }
  return n;
}

// An intermediate frame that does NOT catch. A throw here unwinds straight
// through total() and propagates to its caller — that pass-through is propagation.
static int total(String[] texts) {
  int sum = 0;
  for (String text : texts) {
    sum += parsePositive(text);
  }
  return sum;
}

try {
  // If total() throws, this line is skipped and control jumps to a matching catch.
  int sum = total(batch);
  System.out.println("ok: sum = " + sum);
} catch (NumberFormatException e) {
  // Most specific first: a subclass must precede its superclass below.
  System.out.println("caught NumberFormatException: " + e.getMessage());
} catch (IllegalArgumentException e) {
  System.out.println("caught IllegalArgumentException: " + e.getMessage());
} finally {
  // finally ALWAYS runs — after success or after a catch — so cleanup goes here.
  System.out.println("  (finally: batch done)");
}
