Lazy / infinite list

A lazy list defers computation until a value is actually demanded — the call-by-need evaluation strategy — instead of producing every element up front.
Because nothing is forced before it is needed, the source can be infinite: it describes how to generate the next element rather than holding them all in memory.
Here an unbounded stream of Fibonacci numbers is defined once, then a downstream limit(10) draws exactly ten values through it — the generator runs ten times, never more.

// Java
import java.util.stream.Stream;

// An infinite lazy list of Fibonacci numbers, generated by call-by-need.
// Stream.iterate only records the seed pair and the successor function —
// nothing is computed until a terminal operation pulls values through.
Stream<long[]> fibPairs = Stream.iterate(
    new long[] {0, 1},                            // seed: (F0, F1)
    pair -> new long[] {pair[1], pair[0] + pair[1]} // next: (F1, F0+F1)
);

// Lazy pipeline: map projects each pair to its first element (the Fibonacci
// number). This stage is just a recipe — it runs only as values are demanded.
fibPairs
    .map(pair -> {
      // Visible proof of laziness: this prints exactly 10 times, not infinitely.
      System.out.println("computing Fibonacci number " + pair[0]);
      return pair[0];
    })
    // limit bounds the otherwise-infinite source to the first 10 elements.
    .limit(10)
    // forEach is the terminal operation that drives the pipeline one value at a time.
    .forEach(fib -> System.out.println("got " + fib));
