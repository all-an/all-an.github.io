Event loop

An event loop keeps checking for ready work and runs queued callbacks one turn at a time.
In TypeScript runtimes such as browsers or Node.js, timers, promise callbacks, and I/O completions are scheduled onto queues that the runtime drains in a defined order.
This is why asynchronous tasks can make progress even though your synchronous code appears single-threaded.

// TypeScript
console.log("script start");

setTimeout(() => {
  // Timer callbacks are queued for a later turn of the event loop.
  console.log("timeout callback");
}, 0);

Promise.resolve().then(() => {
  // Promise callbacks run after the current stack, before the timer queue.
  console.log("promise callback");
});

// This runs first because it is still on the current call stack.
console.log("script end");
