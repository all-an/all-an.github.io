Continuation / callback

A callback is a function you hand to another function so it can continue the computation later.
This is one of the oldest async patterns in TypeScript and JavaScript: start work now, then provide the next step as a continuation to run when the result is ready.
Promises and async/await build on this idea, but the callback pattern is the simpler core.

// TypeScript
type User = { name: string };

function loadUser(callback: (user: User) => void): void {
  // Start async work now and remember the continuation for later.
  setTimeout(() => {
    callback({ name: "Ana" });
  }, 100);
}

loadUser(user => {
  // This continuation runs only after loadUser finishes its delayed work.
  console.log(`hello ${user.name}`);
});

// This line runs immediately, before the callback above fires.
console.log("request started");
