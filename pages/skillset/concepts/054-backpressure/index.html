Backpressure

Backpressure is flow control for streams: when a producer emits faster than a consumer can process, the consumer signals how much it can take so the producer never overwhelms it.
In Project Reactor every subscriber requests an explicit amount of demand, and operators such as onBackpressureDrop, onBackpressureBuffer, and onBackpressureLatest decide what happens to values that exceed it.
Here a slow subscriber asks for only three items, so the fast source's remaining values are dropped rather than buffered without bound.

// Java (Project Reactor)
import org.reactivestreams.Subscription;
import reactor.core.publisher.BaseSubscriber;
import reactor.core.publisher.Flux;

// A fast source of 10 values feeding a deliberately slow consumer.
Flux.range(1, 10)
    // Backpressure strategy: discard whatever the consumer has not asked for.
    .onBackpressureDrop(dropped -> System.out.println("dropped  " + dropped))
    .subscribe(new BaseSubscriber<Integer>() {
      @Override
      protected void hookOnSubscribe(Subscription subscription) {
        // Request only 3 items, never more — this bounded demand is the backpressure signal.
        request(3);
      }

      @Override
      protected void hookOnNext(Integer value) {
        // Only the 3 requested values reach us; the overflow is dropped upstream.
        System.out.println("consumed " + value);
      }
    });
