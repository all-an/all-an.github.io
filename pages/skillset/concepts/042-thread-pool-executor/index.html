Thread pool / executor

An executor separates submitting work from deciding where that work runs.
Traditional executors reuse a pool of worker threads, while modern Java can also create one virtual thread per task through an executor API.
The key idea is that callers hand tasks to a runtime component that schedules and runs them.

// Java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;

try (ExecutorService executor = Executors.newFixedThreadPool(2)) {
  // Submit two tasks; the executor decides which worker thread runs each one.
  Future<String> left = executor.submit(() -> "left done");
  Future<String> right = executor.submit(() -> "right done");

  // The main thread waits here until each background task completes.
  System.out.println(left.get());
  System.out.println(right.get());
}
