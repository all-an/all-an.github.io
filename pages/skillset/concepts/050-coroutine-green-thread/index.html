Coroutine / green thread

A coroutine or green thread is a lightweight execution unit that can suspend and resume without the cost of a full platform thread.
In modern Java, Project Loom gives this style through virtual threads, which let blocking code scale much more like coroutine-based systems.
The idea is still sequential code per task, but the runtime can manage a very large number of those tasks efficiently.

// Java
Thread request1 = Thread.startVirtualThread(() -> {
  // This task can block in a direct style without consuming a dedicated OS thread forever.
  System.out.println("request 1 started");
  try {
    Thread.sleep(100);
  } catch (InterruptedException ignored) {
  }
  System.out.println("request 1 finished");
});

Thread request2 = Thread.startVirtualThread(() -> {
  // A second lightweight task runs independently with the same direct coding style.
  System.out.println("request 2 started");
  try {
    Thread.sleep(100);
  } catch (InterruptedException ignored) {
  }
  System.out.println("request 2 finished");
});

// The main thread waits here so both virtual-thread tasks complete before exit.
request1.join();
request2.join();
