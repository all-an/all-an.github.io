Future / deferred value

A future represents a value that is not ready yet, but will appear later after some asynchronous work finishes.
Java commonly uses CompletableFuture, while JavaScript uses Promise for the same basic idea.
The caller starts work now, keeps a handle to the eventual result, and then continues once that result becomes available.

// Java
import java.util.concurrent.CompletableFuture;

// Start asynchronous work and immediately get a handle to its future result.
CompletableFuture<String> message =
    CompletableFuture.supplyAsync(() -> "hello from Java");

// The continuation runs later, after the background computation completes.
CompletableFuture<Void> done = message.thenAccept(value -> {
  System.out.println(value);
});

// Wait here so the program does not exit before the async flow finishes.
done.join();
// JavaScript
// Start asynchronous work and immediately receive a Promise for the future value.
const message = new Promise(resolve => {
  setTimeout(() => resolve("hello from JavaScript"), 100);
});

// This callback runs later, after the Promise is fulfilled.
message.then(value => {
  console.log(value);
});

// Other synchronous code keeps running right away.
console.log("waiting...");
