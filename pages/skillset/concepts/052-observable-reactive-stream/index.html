Observable / reactive stream

An observable, or reactive stream, is a producer that pushes a sequence of values to its subscribers over time, instead of returning a single value once.
It is lazy and composable: operators such as filter and map describe a pipeline that stays idle until something subscribes, and each value then flows through it to the subscriber.
The examples below build the same pipeline — the difference is only the host language and runtime.
