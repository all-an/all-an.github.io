Monad / chain

A monad is a type that wraps a value and knows how to chain steps together with flatMap (also called bind or >>=). Each step produces a new wrapped value, and the monad decides what threading them together means.
For Either it means short-circuiting: the moment a step returns a Left (a failure), the rest of the chain is skipped and that failure becomes the result — no manual if error then return at every step.
Scala's real gift here is the for-comprehension: write the steps linearly and the compiler desugars them straight back into the flatMap chain. The two blocks below are the same program.

// Scala
// Either[L, R] is a monad: it has flatMap (a.k.a. bind / >>=) and map, so a
// failure (Left) short-circuits the rest of the chain.
def parse(text: String): Either[String, Int] =
  text.toIntOption.toRight(s"not a number: '$text'")

def divide(a: Int, b: Int): Either[String, Int] =
  if b == 0 then Left("divide by zero") else Right(a / b)

// Nested flatMap: bind each step to the next. A Left anywhere stops the chain.
val chained: Either[String, Int] =
  parse("84").flatMap(x => parse("2").flatMap(y => divide(x, y)))

// The SAME computation as a for-comprehension — the compiler desugars this
// straight back into the flatMap chain above. Identical result, linear to read.
val sugared: Either[String, Int] =
  for
    x <- parse("84")
    y <- parse("2")
    z <- divide(x, y)
  yield z

println(chained) // Right(42)
println(sugared) // Right(42)
