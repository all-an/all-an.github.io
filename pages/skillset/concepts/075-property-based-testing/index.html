Property-based testing

Example-based tests check a few hand-picked inputs. Property-based testing instead states a rule that must hold for every input, and the framework generates hundreds of random cases trying to break it. When it finds a failure, it shrinks the counterexample down to the smallest input that still fails — so you get [0] rather than some 200-element mess.
Here, jqwik (the Java library, alongside QuickCheck, Hypothesis, ScalaCheck, fast-check) runs each @Property against 1000 generated lists. A property method just returns true when the rule holds. The four below are common shapes: an involution (reverse twice = original), an invariant (size is preserved), a relational property (first becomes last), and an oracle property (agree with a trusted reference). This is a runnable Maven project; see README.md.

// Java + jqwik
// Each @Property is checked against ~1000 randomly generated inputs. If one
// fails, jqwik SHRINKS it to the smallest counterexample before reporting.
class ListsProperties {

  // Involution: reversing twice restores the original list.
  @Property
  boolean reversingTwiceRestoresTheOriginal(@ForAll List<Integer> xs) {
    return Lists.reverse(Lists.reverse(xs)).equals(xs);
  }

  // Invariant: reverse never changes the number of elements.
  @Property
  boolean reversePreservesSize(@ForAll List<Integer> xs) {
    return Lists.reverse(xs).size() == xs.size();
  }

  // Relational, on non-empty lists: the first element ends up last.
  @Property
  boolean firstElementBecomesLast(@ForAll @Size(min = 1) List<Integer> xs) {
    List<Integer> reversed = Lists.reverse(xs);
    return reversed.get(reversed.size() - 1).equals(xs.get(0));
  }

  // Oracle: our reverse agrees with the JDK reference for every input.
  @Property
  boolean matchesReferenceImplementation(@ForAll List<Integer> xs) {
    List<Integer> expected = new ArrayList<>(xs);
    Collections.reverse(expected);
    return Lists.reverse(xs).equals(expected);
  }
}

Want proof it really generates 1000 inputs? DemoProperties in the project uses @Report(Reporting.GENERATED) and a counter to print each generated list and the invocation total — run mvn test -Dtest=DemoProperties. The default count is set per property with @Property(tries = N).

Run the properties yourself with the Maven project in this folder — see README.md.
