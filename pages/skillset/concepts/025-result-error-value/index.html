Result / error value

A result type represents either a successful value or an error value in the type system, instead of signaling failure with exceptions or nulls.
It forces the caller to handle both cases explicitly, which makes control flow around failure more visible.
Java does not have a standard built-in Result type, but Scala does with Either (and Try).

// Scala
// Either[L, R] holds an error on the Left or a value on the Right
def parseAge(text: String): Either[String, Int] =
  if (text == "42") Right(42)
  else Left("invalid age")

val age = parseAge("42")

// Pattern matching forces both cases to be handled
age match {
  case Right(value) => // success: use `value`
  case Left(error)  => // failure: use `error`
}
