C5: Commented-Out Code

Commented-out code is source that has been disabled with comment markers instead of deleted — left behind "just in case" someone needs it again.
Because no one who finds it dares remove it, it gathers like dregs: it drifts out of date, clutters the file, and makes every reader stop to wonder whether it still matters.
The fix is to delete it outright. Version control already remembers every line you ever wrote, so the code is never truly lost — you can always bring it back from history.

Avoid — dead lines kept alive as comments
// Java
public BigDecimal price(Order order) {
    BigDecimal total = order.subtotal();
    // total = total.multiply(LEGACY_TAX_RATE);
    // if (order.isExpedited()) {
    //     total = total.add(RUSH_FEE);
    // }
    total = total.add(order.tax());
    return total;
}
Prefer — delete it; git remembers if you ever need it back
// Java
public BigDecimal price(Order order) {
    BigDecimal total = order.subtotal();
    total = total.add(order.tax());
    return total;
