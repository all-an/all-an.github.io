F4: Dead Function

A dead function is a method that is never called — nothing in the codebase reaches it any more.
It still costs the reader: they have to study it, keep it compiling, and wonder whether it matters, all for code that does nothing because no caller is left to invoke it. Worse, it quietly drifts out of date, so when someone finally does call it they inherit a bug.
The fix is to delete it. Version control keeps the history, so if the need ever returns the code can be recovered — meanwhile the living code is smaller and clearer.

Avoid — keeping a method no one calls
// Java
public class InvoiceService {

    public Money total(Invoice invoice) {
        return invoice.lineItems().stream()
            .map(LineItem::amount)
            .reduce(Money.ZERO, Money::plus);
    }

    // Last caller was removed three refactors ago — now unreachable, but
    // still here being maintained, compiled, and read by everyone.
    private Money totalWithLegacyTax(Invoice invoice) {
        return total(invoice).times(LEGACY_TAX_MULTIPLIER);
    }
}
Prefer — delete it; git remembers if it is ever needed again
