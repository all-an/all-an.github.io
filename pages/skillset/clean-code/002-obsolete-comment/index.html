C2: Obsolete Comment

A comment is obsolete once it no longer matches the code it describes — it has grown old, irrelevant, or simply wrong.
Code changes faster than the comments around it, so stale comments drift away from the truth and become islands of misdirection that mislead the next reader.
The fix is to avoid comments that are likely to age, and to update or delete any obsolete comment the moment you notice it.

Avoid — a comment that no longer matches the code
// Java
// Retries the request up to 3 times before giving up.
public Response fetch(Request request) {
    // The code was later changed to 5 attempts, but the comment was never updated.
    for (int attempt = 1; attempt <= 5; attempt++) {
        Response response = send(request);
        if (response.isOk()) return response;
    }
    throw new RetryExhaustedException(request);
}
Prefer — let a named constant state the fact, so it cannot go stale
// Java
private static final int MAX_ATTEMPTS = 5;

public Response fetch(Request request) {
    for (int attempt = 1; attempt <= MAX_ATTEMPTS; attempt++) {
        Response response = send(request);
        if (response.isOk()) return response;
    }
    throw new RetryExhaustedException(request);
}
