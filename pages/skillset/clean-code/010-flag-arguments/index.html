F3: Flag Arguments

A flag argument is a boolean passed into a function to switch its behaviour — a loud admission that the function does more than one thing.
At the call site the flag is meaningless on its own: render(true) tells the reader nothing, who must open the function to learn what true selects, and the body splits into two tangled paths.
The fix is to split the function into two well-named functions, one per branch, so each does a single thing and the call says what it means.

Avoid — a boolean that picks which behaviour runs
// Java
// Two behaviours wearing one name; the body forks on the flag.
public String render(boolean isSummary) {
    if (isSummary) {
        return renderSummaryView();
    }
    return renderFullView();
}

// Call site: what does `true` mean here? You have to go read render().
String html = render(true);
Prefer — one function per behaviour, each named for what it does
// Java
public String renderSummary() {
    return renderSummaryView();
}

public String renderFull() {
    return renderFullView();
}

// Call site reads as plain intent — no flag to decode.
String html = renderSummary();
