C1: Inappropriate Information

A comment should only carry information that the code itself cannot express — a technical note about design or intent.
It is inappropriate to hold information that belongs in another system: change histories, authorship, last-modified dates, ticket numbers, and TODOs are better kept in version control and your issue tracker.
Such metadata rots quickly, duplicates the source of truth, and adds noise that hides the few comments that actually matter.

Avoid — metadata that belongs elsewhere
// Java
/*
 * Author:        jdoe
 * Created:       2026-05-01
 * Last modified: 2026-05-20 by asmith
 * Change log:
 *   2026-05-10  fixed rounding bug (see JIRA-1234)
 *   2026-05-20  renamed from computeTax
 * TODO: ask Bob whether VAT applies to gift cards
 */
public BigDecimal calculateTax(Order order) {
    return order.subtotal().multiply(TAX_RATE);
}
Prefer — a technical note the code cannot state
// Java
// Rounded HALF_UP so totals match the accounting ledger, which never rounds down.
public BigDecimal calculateTax(Order order) {
    return order.subtotal()
                .multiply(TAX_RATE)
                .setScale(2, RoundingMode.HALF_UP);
}
