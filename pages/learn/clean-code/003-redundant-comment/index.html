C3: Redundant Comment

A comment is redundant when it says nothing the code does not already say for itself — it merely restates the obvious.
Such comments are pure noise: they take longer to read than the code, add no information, and still risk going stale.
The fix is to delete the redundant comment and, when a name or signature already makes the intent clear, let the code speak.

Avoid — comments that just echo the code
// Java
public class Account {
    // The account balance.
    private int balance;

    // Returns the balance.
    public int getBalance() {
        return balance; // return the balance
    }

    // Increment i by 1.
    public void tick() {
        i++;
    }
}
Prefer — let the names carry the meaning, no comment needed
// Java
public class Account {
    private int balance;

    public int getBalance() {
        return balance;
    }

    public void tick() {
        i++;
    }
}
