C4: Poorly Written Comment

A comment worth writing is worth writing well — if you are going to spend the reader's time, make it count.
A poorly written comment rambles, states the obvious, misuses grammar, or buries the one useful fact under noise, so the reader gives up and the comment earns its keep no longer.
The fix is to write few comments, but make each one brief, correct, and clear — say only what the code cannot.

Avoid — rambling, obvious, and imprecise
// Java
// This method here is used for the purpose of getting the value of the
// rate. it returns the rate. note: the rate is a number. dont forget!!!
// (i think this was added a while ago, not totally sure why tbh)
public double getRate() {
    return rate;
}
Prefer — one precise note that the code cannot state, or none at all
// Java
// Annual rate as a fraction (0.05 == 5%); the billing engine expects fractions, not percents.
public double getRate() {
    return rate;
}
