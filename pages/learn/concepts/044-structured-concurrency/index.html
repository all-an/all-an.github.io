Structured concurrency

Structured concurrency treats a group of child tasks as one bounded unit: start them here, wait here, and cancel them together if needed.
In Java, Project Loom exposes this with StructuredTaskScope, usually running subtasks as virtual threads with failure and cancellation rules tied to one lexical block.
This is the Loom-style alternative to STM thinking: instead of transactional shared memory, Java encourages isolating work, joining results in one scope, and keeping locks or atomics only for truly shared state.

// Java
import java.util.concurrent.StructuredTaskScope;

record User(String name) {}

try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
  // Fork two child tasks that run concurrently inside this scope.
  var user = scope.fork(() -> new User("Ana"));
  var orders = scope.fork(() -> java.util.List.of("Book", "Pen"));

  // The parent waits here until both finish, or until one failure shuts the scope down.
  scope.join();
  scope.throwIfFailed(RuntimeException::new);

  // After join succeeds, the parent thread reads both results in one place.
  System.out.println(user.get().name() + " / " + orders.get().size());
} catch (InterruptedException e) {
  Thread.currentThread().interrupt();
}
