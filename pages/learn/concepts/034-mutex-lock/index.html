Mutex / lock

A mutex lets only one thread enter a critical section at a time, preventing conflicting updates to shared state.
Java has built-in synchronization and also explicit lock classes such as ReentrantLock.
The protected code should always unlock in a finally block.

// Java
import java.util.concurrent.locks.ReentrantLock;

ReentrantLock lock = new ReentrantLock();
int[] counter = {0};

lock.lock();
try {
  counter[0]++;
  System.out.println(counter[0]); // 1
} finally {
  lock.unlock();
}
