Effect system

An effect system represents side effects such as I/O as values that can be composed before they are actually executed.
TypeScript does not have this built in, but the idea can be modeled by wrapping work in objects or functions and running them explicitly later.
The key distinction is between describing an effectful program and executing it.

// TypeScript
type Effect<T> = {
  run: () => T;
  map: <U>(fn: (value: T) => U) => Effect<U>;
  flatMap: <U>(fn: (value: T) => Effect<U>) => Effect<U>;
};

const Effect = <T>(run: () => T): Effect<T> => ({
  run,
  map: fn => Effect(() => fn(run())),
  flatMap: fn => Effect(() => fn(run()).run())
});

const succeed = <T>(value: T): Effect<T> => Effect(() => value);
const sync = <T>(thunk: () => T): Effect<T> => Effect(thunk);

// Build the program first; no side effect happens during this composition phase.
const program: Effect<void> = succeed("Ana")
  .flatMap(name => sync(() => `hello ${name}`))
  .flatMap(message => sync(() => console.log(message)));

// The effect runs only here, when the outer program is explicitly executed.
program.run();
