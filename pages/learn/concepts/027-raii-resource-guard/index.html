RAII / resource guard

RAII means a resource is tied to an object or scope so cleanup happens automatically when that scope ends.
In Java this is usually done with try-with-resources, which closes an AutoCloseable even if an exception happens.
It is not full C++-style deterministic destruction, but it gives Java a reliable scoped resource guard.

// Java
import java.io.BufferedReader;
import java.io.FileReader;

try (BufferedReader reader = new BufferedReader(new FileReader("data.txt"))) {
  String line = reader.readLine();
  System.out.println(line);
} // `reader` is closed automatically here
