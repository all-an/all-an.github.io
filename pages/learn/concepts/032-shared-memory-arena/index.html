Shared memory / arena

An arena groups allocations under one lifetime, so many memory regions can be released together instead of one by one.
In Java's Foreign Function & Memory API, an Arena manages off-heap memory segments and can be shared across threads.
This gives Java a scoped way to work with native shared memory outside the garbage-collected heap.

// Java (FFM API)
import java.lang.foreign.Arena;
import java.lang.foreign.MemorySegment;
import java.lang.foreign.ValueLayout;

try (Arena arena = Arena.ofShared()) {
  MemorySegment counter = arena.allocate(ValueLayout.JAVA_INT);
  counter.set(ValueLayout.JAVA_INT, 0, 41);

  Thread worker = Thread.startVirtualThread(() -> {
    int value = counter.get(ValueLayout.JAVA_INT, 0);
    counter.set(ValueLayout.JAVA_INT, 0, value + 1);
  });

  try {
    worker.join();
  } catch (InterruptedException ignored) {
  }
  System.out.println(counter.get(ValueLayout.JAVA_INT, 0)); // 42
}
