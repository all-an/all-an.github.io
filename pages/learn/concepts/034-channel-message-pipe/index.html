Channel / message pipe

A channel sends messages between concurrent tasks instead of sharing mutable state directly.
Java does not have a built-in chan keyword, but a BlockingQueue can play the same role as a message pipe.
One thread puts messages in, another thread takes them out.

// Java
import java.util.concurrent.BlockingQueue;
import java.util.concurrent.LinkedBlockingQueue;

BlockingQueue<String> channel = new LinkedBlockingQueue<>();

Thread producer = Thread.startVirtualThread(() -> {
  try {
    channel.put("hello");
  } catch (InterruptedException ignored) {
  }
});

Thread consumer = Thread.startVirtualThread(() -> {
  try {
    System.out.println(channel.take()); // hello
  } catch (InterruptedException ignored) {
  }
});

try {
  producer.join();
  consumer.join();
} catch (InterruptedException ignored) {
}
