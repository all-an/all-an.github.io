Atomic operation

An atomic operation changes one memory location as one indivisible step, so other threads never observe a half-finished update.
Java exposes this with classes such as AtomicInteger, often implemented with compare-and-swap instead of a lock.
Atomics are great for single-variable coordination, but bigger invariants across multiple fields usually still need locks or other higher-level structure.

// Java
import java.util.concurrent.atomic.AtomicInteger;

AtomicInteger counter = new AtomicInteger(0);

// Each task increments the same counter without taking an explicit lock.
Thread t1 = Thread.startVirtualThread(() -> {
  counter.incrementAndGet();
});

Thread t2 = Thread.startVirtualThread(() -> {
  counter.incrementAndGet();
});

// After both tasks finish, the main thread reads the final atomic value.
t1.join();
t2.join();
System.out.println(counter.get()); // 2
