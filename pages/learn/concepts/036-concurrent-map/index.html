Concurrent map

A concurrent map lets many threads read and update a shared key-value structure without wrapping the whole map in one big lock.
In Java, ConcurrentHashMap uses internal coordination strategies so common operations scale better than a plain HashMap under contention.
It helps for shared registries and caches, but compound workflows across multiple keys may still need extra synchronization.

// Java
import java.util.concurrent.ConcurrentHashMap;

ConcurrentHashMap<String, Integer> counts = new ConcurrentHashMap<>();

// Start two tasks that update the same shared map concurrently.
Thread t1 = Thread.startVirtualThread(() -> {
  counts.merge("apples", 1, Integer::sum);
});

Thread t2 = Thread.startVirtualThread(() -> {
  counts.merge("apples", 1, Integer::sum);
});

// Wait until both updates finish before reading the final value.
t1.join();
t2.join();
System.out.println(counts.get("apples")); // 2
