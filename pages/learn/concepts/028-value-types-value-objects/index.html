Value types / value objects

A value object is a data value without object identity, so two instances are compared by their contents rather than by "who they are".
Project Valhalla adds this idea to Java with value class, aiming for denser layouts and fewer identity-related costs.
This syntax is still a Valhalla preview feature, not regular released Java syntax yet.

// Java (Project Valhalla preview)
value class Point {
  int x;
  int y;

  Point(int x, int y) {
    this.x = x;
    this.y = y;
  }

  Point translate(int dx, int dy) {
    return new Point(x + dx, y + dy);
  }
}

Point a = new Point(10, 20);
Point b = a.translate(0, 0);

System.out.println(a == b); // true: compared by value, not identity
