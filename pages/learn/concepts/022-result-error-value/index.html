Result / error value

A result type represents either a successful value or an error value in the type system, instead of signaling failure with exceptions or nulls.
It forces the caller to handle both cases explicitly, which makes control flow around failure more visible.
Java does not have a standard built-in Result type, but C++23 does with std::expected.

// C++
#include <expected>
#include <string>

std::expected<int, std::string> parseAge(const std::string& text) {
  if (text == "42") return 42;
  return std::unexpected("invalid age");
}

auto age = parseAge("42");

if (age) {
  // success
  int value = *age;
}
