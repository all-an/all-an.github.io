Manual memory management

Manual memory management means the program explicitly allocates and frees memory instead of leaving all of it to the garbage collector.
In GraalVM Native Image, Java can do this with UnmanagedMemory, which works like a low-level malloc/free API for native memory.
This is powerful, but you must release the memory yourself or you leak it.

// Java (GraalVM Native Image)
import org.graalvm.nativeimage.UnmanagedMemory;
import org.graalvm.nativeimage.c.type.CIntPointer;

CIntPointer numbers = UnmanagedMemory.malloc(3 * Integer.BYTES);
// `numbers` is the unmanaged pointer. It points to native memory outside the Java heap.

try {
  numbers.write(0, 10);
  numbers.write(1, 20);
  numbers.write(2, 30);

  int sum = numbers.read(0) + numbers.read(1) + numbers.read(2);
  System.out.println(sum); // 60
} finally {
  UnmanagedMemory.free(numbers); // Manual free is required.
}
